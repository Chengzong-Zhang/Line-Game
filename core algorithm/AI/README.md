# LIFELINE（生命线）课程大作业说明

> 游戏人工智能课程大作业验收入口  
> 选题方向：搜索与规划  
> 核心 AI：Minimax + Alpha-Beta 剪枝

## 1. 游戏简介

LIFELINE（生命线）是一款运行在三角网格上的回合制圈地策略游戏。玩家从各自的初始节点出发，通过落子和自动连线扩张防线；也可以把棋子落在敌方连线上，切断对手与初始节点之间的连接，使失联节点和连线从棋盘上消失。

本项目不是普通的固定规则棋类复刻。连线切断、连通分量删除、领土计算和全局同形禁手会持续改变博弈状态，玩家需要判断扩张收益、连接安全和攻击时机。AI 会依据当前完整棋局主动选择落点，直接参与并影响玩法流程。

项目提供两种可演示的 AI Demo：

- **Web AI Demo（推荐验收）**：浏览器内运行，支持本地双人人机对战、AI 先后手、三档难度和 AI 提示。
- **Pygame AI Demo**：桌面窗口运行，支持人机对战、三档难度、Top-5 候选落点可视化和 AI 提示。

## 2. 核心玩法与规则

### 棋盘与落子

- Pygame Demo 使用边长为 `9` 的三角网格，共 `45` 个格点。
- Web 端支持边长 `6` 至 `15`，并支持双人和三人对局。
- 双人局中，蓝方从 `(0, 0)` 出发，红方从棋盘另一角出发。
- 新节点必须能沿同行、同列或 `x + y` 相同的斜线连接到己方已有节点。
- 新节点会与所有无遮挡、可直线连接的己方节点自动连线。

### 攻击与限制

- 玩家可以落在敌方连线上发动攻击。
- 连线被切断后，无法继续连接到敌方初始节点的节点与连线会被清除。
- 普通扩张不能形成三个两两相邻的己方节点；攻击落子不受该限制。
- 玩家不能落在对方初始节点附近的保护区。
- Superko 全局同形禁手禁止棋局回到任意历史状态，避免无限循环。

### 胜利与失败条件

- 当前玩家没有合法走法时，系统会自动跳过其回合。
- 所有仍在场玩家连续跳过后，对局结束。
- 由己方节点和连线闭合、且内部没有敌方元素的区域视为领土。
- 终局时领土格点数最多的玩家获胜；面积相同则平局。
- Web 三人局还支持认输；只剩一名未认输玩家时，该玩家获胜。

## 3. 玩家如何操作

### Web AI Demo

1. 打开页面后，在棋盘设置中选择 `2` 人。
2. 在“AI 对手”中选择“AI 先手”或“AI 后手”。
3. 选择简单、普通或困难难度。
4. 点击棋盘上的合法格点落子。
5. 使用棋盘下方按钮跳过、重置或请求 AI 提示。
6. AI 回合会显示“AI 思考中”，计算完成后自动落子。

Web AI 只在**本地双人局**运行。联机模式和三人模式不会启动 AI。

### Pygame AI Demo

- 鼠标左键：在合法格点落子。
- `Skip`：主动跳过当前回合。
- 模式选择：按 `1` 选择人人对战，按 `2` 选择人类蓝方对 AI 红方，按 `3` 选择 AI 蓝方对人类红方。
- 难度选择：按 `E`、`M`、`H` 选择搜索深度 `2`、`3`、`4`。
- `Enter` 或空格：开始游戏。
- 游戏中按 `H` 或点击“提示”：显示推荐落点与简短理由。

## 4. 依赖安装与启动

以下命令均从项目根目录执行。项目不调用 LLM 或其他付费 API，不需要配置 API Key。

### 路线 A：运行 Web AI Demo（推荐）

环境要求：

- Python 3.10 或更高版本
- 支持 ES Module、Web Worker 和 Canvas 的现代浏览器
- 首次打开页面时可访问互联网，以加载 Vue 与 MathJax CDN 资源

安装后端依赖：

```powershell
python -m pip install -r ".\web\web后端\requirements.txt"
```

启动 Web 服务：

```powershell
powershell -ExecutionPolicy Bypass -File ".\web\start\start_online_server.ps1"
```

启动脚本会在 `8000` 至 `8004` 中选择可用端口，并自动打开浏览器。进入本地模式后，在棋盘设置中启用 AI 对手即可演示。

### 路线 B：运行 Pygame AI Demo

环境要求：

- Python 3.8 或更高版本
- Pygame

安装依赖：

```powershell
python -m pip install pygame
```

将核心规则目录加入 Python 模块搜索路径：

```powershell
$env:PYTHONPATH = (Resolve-Path ".\core algorithm").Path
```

启动 AI 版本：

```powershell
python ".\core algorithm\AI\main_ai.py"
```

启动不带 AI 的原始双人版本：

```powershell
python ".\core algorithm\triangular_game.py"
```

## 5. AI 机制在玩法中的作用

AI 不是局外分析工具，而是会实际占据一方、推进回合并改变棋局的对手。玩家需要观察 AI 的扩张路线、保护自己的关键连接，并判断是否应主动切断 AI 防线。

AI 的每次决策包含以下过程：

1. 使用完整规则引擎生成合法走法，包括保护区、三点限制和 Superko 检查。
2. 优先搜索能切断敌方连线的攻击走法，其次搜索贴近己方连线的扩张走法。
3. 对候选走法执行 Minimax 对抗搜索。
4. 使用 Alpha-Beta 剪枝跳过不可能成为最优选择的分支。
5. 使用启发式评估函数比较搜索叶节点，选择分值最高的落点。

评估函数综合考虑：

| 评估项 | 作用 |
| --- | --- |
| 节点数量优势 | 衡量双方可持续扩张的基础 |
| 节点与连线覆盖优势 | 衡量当前盘面控制范围 |
| BFS 领土近似 | 快速估计双方可达空间 |
| 攻击威胁数量 | 鼓励寻找可切断敌方连线的机会 |
| 初始节点连通质量 | 避免形成脆弱或失联结构 |

为控制搜索规模，每个搜索节点最多继续分析排序后的前 `20` 个候选走法。

## 6. 为什么选择 Minimax + Alpha-Beta

本游戏是回合制、确定性、完备信息的对抗游戏，没有随机事件和隐藏信息。玩家的扩张会产生收益，但也可能暴露可被切断的连接，因此只评估当前一步并不足以得到合理策略。

Minimax 适合模拟“我方选择最佳行动、对方也会选择最佳反制”的过程；
Alpha-Beta 剪枝和走法排序可以在不改变搜索结果的前提下减少需要展开的分支。
可调搜索深度则能直观体现 AI 决策质量与计算时间之间的权衡，适合作为课程 Demo 展示。

精确领土算法需要执行轮廓追踪、贪心修剪和泛洪计算，直接放入每个搜索节点会显著拖慢 AI。因此搜索评估使用快速 BFS 近似，真实落子与终局计分仍由完整规则引擎计算。

## 7. 可观察的 AI 行为

### Web 端

- AI 先手或后手可切换。
- 搜索深度 `2`、`3`、`4` 可调。
- AI 在独立 Web Worker 中搜索，避免阻塞页面交互与 Canvas 渲染。
- AI 回合显示思考状态。
- 本地玩家可启用提示功能，查看 AI 推荐落点。

### Pygame 端

- AI 在独立线程中计算，主界面保持刷新。
- AI 落子后短暂显示 Top-5 候选点：
  - 黄色星标：最佳候选。
  - 橙色圆圈：第 2 至第 3 候选。
  - 灰色圆圈：第 4 至第 5 候选。
- 玩家可请求 AI 提示，推荐点会以绿色脉冲圆显示，并给出简短理由。

这些功能使 AI 的搜索结果、难度差异和策略选择可以直接被观察，便于课程演示。

## 9. 核心算法与工程设计

- **双层状态模型**：网格物理层负责棋盘状态与渲染，逻辑边集合负责节点连通关系。
- **BFS 连通分量清理**：攻击切断连线后，删除无法连接到初始节点的敌方结构。
- **领土计算**：右手摸墙获取外轮廓，动态贪心修剪边界，再通过泛洪法判断真实覆盖区域。
- **Superko 状态哈希**：将非空格点、逻辑边和下一行动方序列化，禁止历史局面重现。
- **搜索状态快照**：Minimax 展开前保存棋盘、逻辑边、历史哈希、回合和跳过状态，递归结束后完整恢复。
- **非阻塞 AI**：Pygame 使用后台线程，Web 使用 Web Worker。
