# 【任务目标】
你是一个资深的前端工程师和 UX 交互设计师。我们需要对目前基于 Vue3 + Canvas 的网页版棋盘游戏进行 UI 布局重构与交互逻辑优化。请仔细阅读以下需求，并按模块逐步为我输出修改代码。

## 一、 界面布局重构 (UI/UX Restructuring)
目前网页端信息过于拥挤，且移动端体验不佳。需要进行以下层级与布局调整：

1. **左侧固定侧边栏 (Sidebar)：** - 将“对局设置”、“模式选择”、“账号功能”统一收纳至左侧边栏。
   - 使用折叠面板或按钮弹窗的形式展示具体设置，避免占用过多屏幕空间。去除目前重复显示了两次的“对局设置”标题。
2. **核心信息区高亮 (主视觉区)：**
   - 屏幕布局优先级应严格按照：**棋盘状态 (得分/领地) > 本地操作按钮 > 对局设置**。
   - 左侧原有留白区域用来放置“棋盘状态”（关联性最强）。
   - **Bug修复：** 确保在“双人模式”下，计分板中不要错误显示第三人的“紫色领地”信息。
3. **移动端适配：**
   - 针对手机端重新排版，确保紧凑但清晰。必须保证“棋盘状态”和“本地操作”在首屏最显眼的位置，绝对不能让关键操作沉底导致看不见。

## 二、 局内交互体验增强 (In-Game UX)
1. **视觉提示补全：**
   - 增加“最新落子”的高亮提示（例如在最后一颗棋子上加个小光圈）。
   - 增加“对方跳过 (Skip)”的明显 UI 提示。
2. **本地操作扩充：**
   - 增加“读秒 (倒计时)”功能，提升对局紧张感。
   - 在本地操作区新增“认输”按钮。点击后可直接结算，或弹出对话框询问对方“是否同意 (1/2)”。

## 三、 房间与场间系统完善 (Lobby & Inter-Game Logic)
场间（Game Over 后或新开局前）的菜单只有“开启下一局”过于单调且容易产生误解，需要重构为“双向交互”的房间大厅：

1. **准备与开始逻辑：**
   - “开启下一局”必须改为双方交互式的“准备 (Ready)”机制。只有双方都点击同意，才能开始倒计时开局。
   - 场间需要实时显示“对方状态”（例如：准备中/发呆中）。
2. **房间控制权 (房主特权)：**
   - 房主可以直接在房间内设置/切换“先手/后手”。
   - 房主可以切换对局设置。**注意：** 一旦切换对局设置，房间内所有人的“准备”状态必须重置为“未准备”。
3. **新增基础房间选项：**
   - 增加“关闭提示框”和“退出房间”按钮。
